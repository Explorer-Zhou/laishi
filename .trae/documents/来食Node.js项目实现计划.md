# "来食"项目实现计划（简化版）

## 项目结构
创建Node.js全栈项目，包含后端API和React前端，无需用户认证

## 后端实现
1. **项目初始化**
   - 创建Express.js后端项目
   - 配置SQLite数据库
   - 设置环境变量配置

2. **核心服务层**
   - LLM推荐服务：使用OpenAI兼容API，基于用户偏好和营养数据推荐食物
   - 营养分析服务：记录和分析营养摄入数据
   - 高德地图服务：集成高德MCP，根据食物推荐查找附近店铺

3. **数据模型**
   - 用户模型（名称、偏好、饮食限制）
   - 营养记录模型（关联用户名称）
   - 食物推荐模型

4. **API路由**
   - 用户管理（创建用户、获取信息、更新偏好）
   - 营养记录与分析
   - 食物推荐
   - 附近店铺查询

## 前端实现
1. **React应用初始化**
   - 使用Vite创建React项目
   - 配置路由和状态管理

2. **页面组件**
   - 用户设置页：输入用户名称
   - 偏好设置页：管理口味偏好、饮食限制
   - 首页：显示今日推荐
   - 营养分析页：展示营养摄入趋势
   - 食品推荐页：展示推荐食物及附近提供该食物的店铺

3. **API集成**
   - 调用后端API获取推荐和店铺数据
   - 展示店铺信息（名称、地址、评分、距离）

## 关键功能
- 用户名称设置（无需登录）
- 智能食物推荐：结合用户偏好和营养数据
- 营养分析：追踪和展示营养摄入
- 店铺推荐：基于食物推荐展示附近店铺
- 用户偏好管理：个性化推荐基础
